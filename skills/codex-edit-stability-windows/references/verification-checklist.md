# Verification Checklist

## 編集後に確認すること

### 文字化け

- `�` が入っていない
- 意図しない `?` 連続が入っていない
- URL やバッククォート周辺が壊れていない

### BOM / 不可視文字

- `.cs` と `.yaml` は BOM なしを優先
- 先頭不可視文字が混入していない

### 内容

- プレースホルダが意図せず壊れていない
- 置換対象以外が崩れていない
- Markdown 見出しやコードブロックが閉じている

### Git

- 想定外ファイルが変更されていない
- ステージ前に差分対象を確認する

## 簡易チェック例

- 置換文字検索
- 代表ファイルの再読込
- `git status --short`
