# Safe Edit Workflow

## 基本手順

### 1. ファイル種別を確認する

- `.cs`: ASCII 優先、BOM なし推奨
- `.yaml`: ASCII 優先、BOM なし推奨
- `.md`: UTF-8、日本語可、編集後検証必須

### 2. まず `apply_patch` を試す

- 通るならそのまま使う
- 失敗が環境由来なら深追いしない

### 3. `apply_patch` が失敗した場合

- 小さなファイルは UTF-8 明示で全書き換え
- 大きなファイルは安全に読んで置換し、最後に UTF-8 で明示保存
- PowerShell の複雑な文字列置換は避ける

### 4. PowerShell を使う場合のルール

- 文字列中にバッククォートが多い文書は避ける
- 可能なら単純なファイル丸ごと書き換えにする
- 書き込み時は UTF-8 を明示する
- `.cs` や `.yaml` では BOM を避ける方法を選ぶ

### 5. 日本語を含むファイルのルール

- Markdown は日本語でよい
- コードファイルはコメントを英語寄りにするか、必要最小限にする
- 文字化け確認を省略しない

## 推奨順

1. `apply_patch`
2. 安全なテンプレートコピー
3. UTF-8 明示の再書き込み

## 非推奨

- 複数段の PowerShell 文字列置換
- 展開付き here-string による Markdown 再生成
- 未検証のまま `git add`
